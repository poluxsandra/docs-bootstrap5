Visão Geral

Existem várias maneiras de personalizar o Bootstrap. O melhor caminho dependerá do projeto em si, da complexidade de suas ferramentas de compilação, da versão do Bootstrap que você está usando, do suporte ao navegador e muito mais.

Nossos dois métodos preferidos são:

  1. Usando Bootstrap via gerenciador de pacote para que você possa usar e estender nossos arquivos de origem.
  2. Usando os arquivos de distribuição compilados do Bootstrap ou via jsDelivr para que você possa adicionar ou substituir os estilos do Bootstrap.

Embora não possamos entrar em detalhes aqui sobre como usar cada gerenciador de pacotes, podemos fornecer algumas orientações sobre como usar o Bootstrap com seu próprio compilador Sass.

Para aqueles que desejam usar os arquivos de distribuição, revise a página de Introdução da seção Começando para saber como incluir esses arquivos e ter um exemplo de página HTML. A partir daí, consulte os documentos para o layout, componentes e comportamentos que você gostaria de usar.

Conforme você se familiariza com o Bootstrap, continue explorando esta seção para obter mais detalhes sobre como utilizar nossas opções globais, fazendo uso e alterando nosso sistema de cores, como construímos nossos componentes, como usar nossa lista crescente de propriedades personalizadas CSS e como otimizar seu código ao construir com Bootstrap.

CSPs e SVGs incorporados

Vários componentes do Bootstrap incluem SVGs incorporados em nosso CSS para estilizar os componentes de forma consistente e fácil em navegadores e dispositivos. Para organizações com configurações CSP mais estritas, documentamos todas as instâncias de nossos SVGs incorporados (todos dos quais a propriedade background-image pode ser aplicada) para que você possa rever mais detalhadamente suas opções.

  • Acordões
  • Botão de Fechar (usado em alertas e moldais)
  • Botões check e radio de Formulários
  • Mudanças de Formulário
  • Ícones de Validação de Formulário
  • Selecione os Menus
  • Controles de Carrossel
  • Botões de alternância da barra de navegação

Baseado nas conversas com a comunidade, algumas opções para resolver isso em sua própria base de código incluem substituir os URLs por ativos hospedados localmente, remover as imagens e usar imagens embutidas (não é possível em todos os componentes) e modificar seu CSP. Nossa recomendação é revisar cuidadosamente suas próprias políticas de segurança e decidir sobre o melhor caminho a seguir, se necessário.
