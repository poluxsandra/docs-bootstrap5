Otimização

Mantenha seus projetos enxutos, responsivos e fáceis de manter, para que você possa oferecer a melhor experiência e se concentrar em trabalhos mais importantes.

Importações Lean Sass

Ao usar o Sass em seu pipeline de ativos, certifique-se de otimizar o Bootstrap apenas com @import os componentes necessários. Suas maiores otimizações provavelmente virão da seção Layout & Components do nosso bootstrap.scss. 

// Configuração
  @import "functions";
  @import "variables";
  @import "mixins";
  @import "utilities";
  
  // Layout & Componentes
  @import "root";
  @import "reboot";
  @import "type";
  @import "images";
  @import "containers";
  @import "grid";
  @import "tables";
  @import "forms";
  @import "buttons";
  @import "transitions";
  @import "dropdown";
  @import "button-group";
  @import "nav";
  @import "navbar";
  @import "card";
  @import "accordion";
  @import "breadcrumb";
  @import "pagination";
  @import "badge";
  @import "alert";
  @import "progress";
  @import "list-group";
  @import "close";
  @import "toasts";
  @import "modal";
  @import "tooltip";
  @import "popover";
  @import "carousel";
  @import "spinners";
  @import "offcanvas";
  
  // Helpers
  @import "helpers";
  
  // Utilities
  @import "utilities/api";

Se você não estiver usando um componente, comente-o ou exclua-o totalmente. Por exemplo, se você não estiver usando o carrossel, remova essa importação para salvar o tamanho do arquivo no CSS compilado. Lembre-se de que existem algumas dependências nas importações do Sass que podem dificultar a omissão de um arquivo.

JavaScript Enxuto

O JavaScript do Bootstrap inclui todos os componentes em nossos arquivos dist primários (bootstrap.js e bootstrap.min.js), e até mesmo nossa dependência primária (Popper) com nossos arquivos de pacote (Popper) com nossos arquivos de pacote (bootstrap.bundle.js e bootstrap.bundle.min.js). Enquanto estiver personalizando pelo Sass, certifique-se de remover o JavaScript relacionado.

Por exemplo, supondo que você esteja usando seu próprio agrupador de JavaScript, como Webpack ou Rollup, você importaria apenas o JavaScript que planeja usar. No exemplo abaixo, mostramos como incluir apenas nosso JavaScript modal: 

// Importe apenas o que precisa
  
  // import 'bootstrap/js/dist/alert';
  // import 'bootstrap/js/dist/button';
  // import 'bootstrap/js/dist/carousel';
  // import 'bootstrap/js/dist/collapse';
  // import 'bootstrap/js/dist/dropdown';
  import 'bootstrap/js/dist/modal';
  // import 'bootstrap/js/dist/popover';
  // import 'bootstrap/js/dist/scrollspy';
  // import 'bootstrap/js/dist/tab';
  // import 'bootstrap/js/dist/toast';
  // import 'bootstrap/js/dist/tooltip';

Dessa forma, você não está incluindo nenhum JavaScript que não pretende usar para componentes como botões, carrosséis e dicas de ferramentas. Se você estiver importando menus suspensos, dicas de ferramentas ou popovers, certifique-se de listar a dependência de Popper em seu arquivo package.json.

Exportações Padrão

Os arquivos em uso no bootstrap/js/dist são a exportação padrão, portanto, se quiser usar um deles, você deve fazer o seguinte: 

import Modal from 'bootstrap/js/dist/modal'
  
  const modal = new Modal(document.getElementById('myModal'))

Autoprefixador .browserslistrc

O bootstrap depende do Autoprefixador para adicionar automaticamente os prefixos do navegador a certas propriedades CSS. Os prefixos são ditados por nosso arquivo .browserslistrc, encontrado na raiz do repositório Bootstrap. Personalizar esta lista de navegadores e recompilar o Sass removerá automaticamente algum CSS do CSS compilado, se houver prefixos de fornecedor exclusivos para aquele navegador ou versão.

Minificar e Compactar

Sempre que possível, certifique-se de compactar todo o código que você veicula para seus visitantes. Se você estiver usando arquivos dist do Bootstrap, tente usar as versões reduzidas (indicadas pelas extensões .min.css e .min.js). Se você estiver construindo Bootstrap a partir do código-fonte com seu próprio sistema de construção, certifique-se de implementar seus próprios minificadores para HTML, CSS e JS.

Arquivos Não Bloqueadores

Embora minimizar e usar compactação possa parecer suficiente, tornar seus arquivos não bloqueadores também é um grande passo para tornar seu site bem otimizado e rápido o suficiente.

Se você estiver usando um plugin do Lighthouse no Google Chrome, pode ter tropeçado no FCP. A métrica The First Contentful Paint mede o tempo desde o início do carregamento da página até o momento em que qualquer parte do conteúdo da página é renderizada na tela.

Você pode melhorar o FCP adiando JavaScript ou CSS não crítico. O que isso significa? Simplesmente, JavaScript ou folhas de estilo que não precisam estar presentes na primeira pintura de sua página devem ser marcados com atributos async ou defer.

Isso garante que os recursos menos importantes sejam carregados posteriormente e não bloqueiem a primeira pintura. Por outro lado, recursos críticos podem ser incluídos como scripts ou estilos embutidos.

Se você quiser saber mais sobre isso, já existem muitos artigos excelentes sobre isso:

  • https://web.dev/render-blocking-resources/
  • https://web.dev/defer-non-critical-css/

Sempre Use HTTPS

Seu site deve estar disponível apenas em conexões HTTPS em produção. O HTTPS melhora a segurança, a privacidade e a disponibilidade de todos os sites, e não existe tráfego não confidencial na web. As etapas para configurar seu site para ser servido exclusivamente por HTTPS variam amplamente, dependendo de sua arquitetura e provedor de hospedagem na web e, portanto, estão além do escopo destes documentos.

Os sites atendidos por HTTPS também devem acessar todas as folhas de estilo, scripts e outros ativos por meio de conexões HTTPS. Caso contrário, você estará enviando aos usuários conteúdo ativo misto, levando a vulnerabilidades potenciais em que um site pode ser comprometido pela alteração de uma dependência. Isso pode levar a problemas de segurança e avisos no navegador exibidos aos usuários. Esteja você obtendo Bootstrap de um CDN ou servindo-o você mesmo, certifique-se de acessá-lo apenas por meio de conexões HTTPS.
