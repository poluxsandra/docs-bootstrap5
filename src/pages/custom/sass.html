Estrutura de Arquivos

Sempre que possível, evite modificar os arquivos principais do Bootstrap. Para o Sass, isso significa criar sua própria folha de estilo que importa o Bootstrap para que você possa modificar e estendê-lo. Supondo que você está usando um gerenciador de pacotes como o npm, você terá uma estrutura de arquivo semelhante a esta: 

seu-projeto/
├── scss
│   └── custom.scss
└── node_modules/
    └── bootstrap
        ├── js
        └── scss

Se você baixou nossos arquivos-fonte e não está usando um gerenciador de pacotes, você vai querer configurar a folha de estilo manualmente, então o sistema de arquivos teria uma estrutura semelhante a abaixo, mantendo os arquivos-fonte do Bootstrap separados dos seus. 

seu-projeto/
├── scss
│   └── custom.scss
└── bootstrap/
    ├── js
    └── scss

Importando

Em seu custom.scss, você importará os arquivos-fontes Sass do Bootstrap. Você tem duas opções: incluir todo o Bootstrap ou escolher as partes de que precisa. Nós encorajamos o último, embora exista alguns requisitos e dependências em nossos componentes. Você também poderá incluir algum JavaScript em nosso plugin. 

// Custom.scss
// Opção A: Incluir todo o Bootstrap

@import "../node_modules/bootstrap/scss/bootstrap";

// Adicione seu código personalizado aqui

// Custom.scss
// Opção B: Incluir partes do Bootstrap

// Requirido
@import "../node_modules/bootstrap/scss/functions";
@import "../node_modules/bootstrap/scss/variables";
@import "../node_modules/bootstrap/scss/mixins";

// Inclua substituições ao padrão de variáveis personalizadas aqui

// Opcional
@import "../node_modules/bootstrap/scss/root";
@import "../node_modules/bootstrap/scss/reboot";
@import "../node_modules/bootstrap/scss/type";
@import "../node_modules/bootstrap/scss/images";
@import "../node_modules/bootstrap/scss/containers";
@import "../node_modules/bootstrap/scss/grid";

Com essa configuração, você pode começar a modificar qualquer uma das variáveis e mapas Sass em seu custom.scss. Você também pode começar a adicionar partes do Bootstrap na seção //Opcional conforme necessário. Nós sugerimos a pilha de importação completa através do nosso arquivo bootstrap.scss como ponto de partida.

Variáveis Padrões

Cada variável Sass no Bootstrap inclui o sinalizador !default sinalizador, permitindo que você substitua o valor da variável em seu próprio Sass sem modificar o código-fonte do Bootstrap. Copie e cole variáveis conforme necessário, modifique seus valores e remova o sinalizador !default. Se um valor já foi atribuído a uma variável, ele não será reatribuído a uma variável pelos valores padrão no Bootstrap.

Você encontrará uma lista completa das variáveis do Bootstrap no arquivo scss/_variables.scss. Algumas variáveis possui null como valor, essas variáveis não geram nenhuma propriedade a menos que sejam substituídas em sua configuração.

Substituições de variáveis devem vir após nossas funções, variáveis e mixins serem importados, mas antes do resto das importações.

Aqui está um exemplo que altera as propriedades background-color e color do elemento <body> quano se importa e compila o Bootstrap via npm: 

// Requirido
@import "../node_modules/bootstrap/scss/functions";
@import "../node_modules/bootstrap/scss/variables";
@import "../node_modules/bootstrap/scss/mixins";

// Sua variável substitui a padrão
$body-bg: #000;
$body-color: #111;

// Bootstrap e suas variáveis padrão

// Opcional
@import "../node_modules/bootstrap/scss/root";
@import "../node_modules/bootstrap/scss/reboot";
@import "../node_modules/bootstrap/scss/type";
// etc

Repita conforme necessário para qualquer variável no Bootstrap, incluindo as opções globais abaixo.

Comece com Bootstrap via npm com nosso projeto inicial! Vá para o repositório de modelos twbs/bootstrap-npm-starter para ver como construir e personalizar bootstrap em seu próprio projeto npm. Inclui compilador Sass, Autoprefixador, Stylelint, PurgeCSS, e Ícones do Bootstrap.

Mapas e Loops

O Bootstrap inclui um punhado de mapas Sass, pares de valores-chave que tornam mais fácil gerar famílias de CSS relacionadas. Usamos mapas Sass para nossos núcleos, pontos de interrupção de grau e muito mais. Assim como as variáveis Sass, todos os mapas Sass incluem um sinalizador !default e podem ser substituídos e estendidos.

Alguns de nossos mapas Sass são fundidos com mapas vazios por padrão. Isso é feito para permitir a fácil expansão de um determinado mapa Sass, mas remover itens de um mapa fica um pouco mais difícil.

Modificar Mapas

Todos as variáveis do mapa $theme-colors são definidas como variáveis independentes. Para modificar uma cor existente em nosso mapa $theme-colors, no seu arquivo Sass personalizado, adicione o seguinte: 

$primary: #0074d9;
$danger: #ff4136;

Posteriormente, essas variáveis são definidas no mapa $theme-colors do Bootstrap: 

$theme-colors: (
  "primary": $primary,
  "danger": $danger
);

Adicionar ao Mapa

Adicione novas cores ao $theme-colors, ou qualquer outro mapa, criando um novo mapa Sass com seus valores personalizados e mesclando-o com o mapa original. Nesse caso, criaremos um novo mapa chamado de $custom-colors e mesclamos com o mapa $theme-colors. 

// Criação do seu próprio mapa
$custom-colors: (
  "custom-color": #900
);

// Mesclando os mapas
$theme-colors: map-merge($theme-colors, $custom-colors);

Removendo do Mapa

Para remover cores do mapa $theme-colors, ou qualquer outro mapa, use o map-remove. Esteja ciente de que você deve inseri-lo entre os recursos requiridos e os opcionais: 

// Requiridos
@import "../node_modules/bootstrap/scss/functions";
@import "../node_modules/bootstrap/scss/variables";
@import "../node_modules/bootstrap/scss/mixins";

$theme-colors: map-remove($theme-colors, "info", "light", "dark");

// Opcional
@import "../node_modules/bootstrap/scss/root";
@import "../node_modules/bootstrap/scss/reboot";
@import "../node_modules/bootstrap/scss/type";
// etc

Chaves Requiridas

O Bootstrap assume a presença de algumas chaves específicas dentro dos mapas Sass que é usada e extendida pela própria equipe do Bootstrap. À medida que você personaliza os mapas incluídos, você pode encontrar erros onde essas chaves está sendo usada em um mapa Sass específico.

Por exemplo, nós usamos as chavesprimary, success, e danger do mapa $theme-colors para links, botões, e estados de formulário. Substituir os valores dessas chaves não deve apresentar problemas, mas removê-los pode causar problemas de compilação do Sass. Nestes casos, você precisará modificar o código Sass que faz uso desses valores.

Funções

Cores

Ao lado do Mapas Sass nós temos, cores temáticas também que podem ser usadas como variáveis autônomas, como $primary. 

.custom-element {
  color: $gray-100;
  background-color: $dark;
}

Você pode clarear ou escurecer cores com Bootstrap usando as funções tint-color() e shade-color(). Essas funções misturarão cores com preto ou branco, ao contrário das funções nativas do Sass (lighten() e darken()) que iriam alterar a luminosidade em um valor fixo, o que muitas vezes não leva ao efeito desejado. 

// Matizar uma cor: misturar uma cor com branco
@function tint-color($color, $weight) {
  @return mix(white, $color, $weight);
}

// Sombrear uma cor: misture uma cor com preto
@function shade-color($color, $weight) {
  @return mix(black, $color, $weight);
}

// Sombrear a cor se o weight for positivo, senão matiza-lo
@function shift-color($color, $weight) {
  @return if($weight > 0, shade-color($color, $weight), tint-color($color, -$weight));
}

Na prática, você chama a função e passa os parâmetros de cor e weight. 

.custom-element {
  color: tint-color($primary, 10%);
}

.custom-element-2 {
  color: shade-color($danger, 30%);
}

Contraste de Cor

A fim de atender os Padrões de Acessibilidade de Contraste de Cor do WCAG 2.0, autores devem providenciar uma relação de contraste de pelo menos de 4,5:1, com poucas exceções.

Uma função adicional que incluímos no Bootstrap é a função de contraste de cores, color-contrast. Ele utiliza o algoritmo WCAG 2.0 para calcular limites de contraste com base em luminância relativa em um espaço de cor sRGB para retornar automaticamente uma cor de contraste claro (#fff), escuro (#212529) ou preto (#000) baseado na cor especificada. Esta função é especialmente útil para mixins ou loops onde você está gerando várias classes.

Por exemplo, para gerar amostras de cores a partir do nosso mapa $theme-colors: 

@each $color, $value in $theme-colors {
  .swatch-#{$color} {
    color: color-contrast($value);
  }
}

Também pode ser usado para necessidades únicas de contraste: 

.custom-element {
  color: color-contrast(#000); // retorna "color: #fff"
}

Você também pode especificar uma cor de base com nossas funções de mapa de cores: 

.custom-element {
  color: color-contrast($dark); // retorna "color: #fff"
}

Escape SVG

Nós usamos a função escape-svg para escapar os caracteres <, > e # para planos de fundo de imagens SVG. Quando se usa a função escape-svg, os URIs de dados devem ser citados.

Funções de Adicionar e Subtrair

Nós usamos as funções add e subtract para empacotar a função calc no CSS. O objetivo principal dessas funções é evitar erros quando um valor 0 "sem unidade" é passado dentro de uma expressão no calc. Expressões como calc(10px - 0) retornará erro em todos os navegadores, apesar de ser matematicamente correto.

Exemplo de onde o calc é válido: 

$border-radius: .25rem;
$border-width: 1px;

.element {
  // A saída calc (.25rem - 1px) é válida
  border-radius: calc($border-radius - $border-width);
}

.element {
  // Retorna o mesmo calc (.25rem - 1px) como acima
  border-radius: subtract($border-radius, $border-width);
}

Exemplo de quando o calc é inválido: 

$border-radius: .25rem;
$border-width: 0;

.element {
  // A saída calc (.25rem - 0) é inválida
  border-radius: calc($border-radius - $border-width);
}

.element {
  // Saída .25rem
  border-radius: subtract($border-radius, $border-width);
}
