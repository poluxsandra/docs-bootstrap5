Ferramentas de Construção

Aprenda como usar os scripts npm incluídos no Bootstrap para construir nossos documentos, compilar o código-fonte, executar testes e muito mais.

Configuração de Ferramentas

O Bootstrap usa scripts npm para seu sistema de construção. Nosso package.json inclui métodos convenientes para trabalhar com nossa estrutura, incluindo compilação de código, execução de testes e muito mais.

Para usar nosso sistema de construção e rodar a documentação localmente, você precisa copiar os arquivos-fonte do Bootstrap e Node. Faça os seguintes passos:

  1. Baixe e instale o Node.js, ele serve para gerenciar nossas dependências.
  2. Baixe os arquivos-fontes do Bootstrap ou clone o Repositório.
  3. Navegue até a pasta raiz/bootstrap e execute o comando npm install para instalar nossas dependências locais listadas no package.json.

Quando concluído, você poderá executar os vários comandos fornecidos a partir da linha de comando.

Using npm scripts

Nosso package.json inclui inúmeras tarefas para o desenvolvimento do projeto. Execute npm run para ver todos os scripts npm em seu terminal. As tarefas primárias incluem:

Tarefa Descrição
npm start Compila CSS e JavaScript, constrói a documentação e inicia um servidor local.
npm run dist Cria o diretório dist/ com os arquivos compilados. Requer Sass, Autoprefixador, e terser.
npm test Executa testes localmente após executar o comando npm run dist
npm run docs-serve Constrói e executa a documentação localmente.

Comece com Bootstrap via npm com o nosso projeto inicial! Vá para o twbs/bootstrap-npm-starter repositório de modelos para ver como construir e personalizar Bootstrap em seu próprio projeto npm. Incluí o compilador Sass, Autoprefixador, Stylelint, PurgeCSS, e Ícones Bootstrap.

Sass

Bootstrap usa o Dart Sass para compilar nossos arquivos-fonted Sass em arquivos CSS (incluído em nosso processo de construção), e recomendamos que você faça o mesmo se estiver compilando o Sass usando seu próprio pipeline de ativos. Anteriormente, usamos o Node Sass para Bootstrap v4, mas LibSass e pacotes construídos em cima dele, incluindo Node Sass, são agora obsoletos.

Dart Sass usa uma precisão de 10 em rounding e por razões de eficiência não permite ajuste desse valor. Nós não baixamos essa precisão durante o processamento adicional do nosso CSS gerado, nem durante o processo de mimificação, recomendamos manter uma precisão de pelo menos 6 para evitar problemas com rounding do navegador.

Autoprefixador

Bootstrap usa Autoprefixador (incluído no nosso processo de compilação) para adicionar automaticamente prefixos de fornecedor a algumas propriedades CSS no momento da compilação. Isso nos economiza tempo e código, permitindo-nos escrever partes importantes de nosso CSS uma única vez, ao mesmo tempo que elimina a necessidade de mixins de fornecedores como os encontrados na v3.

Mantemos uma lista de navegadores suportados pelo Autoprefixador em um arquivo separado em nosso repositório GitHub. Consulte .browserslistrc para mais detalhes.

RTLCSS

Bootstrap usa RTLCSS para processar CSS compilado e convertê-lo em RTL - basicamente substituindo como propriedades que reconhecem a direção horizontal invertida (por exemplo padding-left) por seus opostos. Ele nos permite apenas escrever nosso CSS uma única vez e fazer pequenos ajustes usando o controle RTLCSS e valores de diretivas.

Documentação Local

Rodar nossa documentação localmente requer o uso do Hugo, que é instalado por meio do pacote npm hugo-bin. Hugo é um gerador de sites estáticos extremamente rápido e extensível que nos fornece: inclusões básicas, arquivos baseados em Markdown, modelos e muito mais. Veja como começar:

  1. Execute as configurações de ferramentas acima, para instalar todas as nossas dependências.
  2. Vai para o diretório-raiz /bootstrap e execute o comando npm run docs-serve.
  3. Abra http://localhost:9001/ no seu navegador, e pronto.

Aprenda mais de como usar o Hugo lendo sua documentação.

Solução de problemas

Se você encontrar problemas ao instalar dependências, desinstalar todas as versões anteriores das dependências (global e local). Então, execute novamente npm install.
