RTL

Aprenda como ativar o suporte para texto da direita para esquerda no Bootstrap em nossos layout, componentes e utilitários.

Familiarizando

Recomendamos familiarizar-se com o Bootstrap primeiro lendo nossa página de introdução dessa seção. Após isto, segue o conteúdo a seguir sobre RTL.

Você também pode querer ler sobre o Projeto RTLCSS, ele reforça nossa abordagem para RTL.

Recurso Experimental

O recurso RTL ainda é experimental e provavelmente irá evoluir de acordo com os feedbacks do usuários. Identificou algo ou tem uma sugestão de melhoria? Abra um issue, adoraríamos receber seus insights.

HTML Requirido

Existem dois requisitos estritos para habilitar RTL em páginas com Bootstrap.

  1. Definir dir="rtl" no elemento <html>.
  2. Adicionar um valor apropriado ao atributo lang, como por exemplo, lang="ar", no elemento <html>.

A partir daí, você precisará incluir uma versão RTL de nosso CSS. Por exemplo, aqui está a folha de estilo para nosso CSS compilado e minimizado com RTL habilitado: 

<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/bootstrap@5.0.0-beta2/dist/css/bootstrap.rtl.min.css" integrity="sha384-4dNpRvNX0c/TdYEbYup8qbjvjaMrgUPh+g4I03CnNtANuv+VAvPL6LqdwzZKV38G" crossorigin="anonymous">

Exemplos de RTL

Comece com um de nossos vários exemplos de RTL.

Abordagem

Nossa abordagem para construir suporte RTL no Bootstrap vem com duas decisões importantes que afetaram como escrevemos e usamos nosso CSS:

  1. Primeiro, decidimos construí-lo com o projeto RTLCSS. Isso nos dá alguns recursos poderosos para gerenciar alterações e substituições ao mudar de LTR para RTL. Também nos permite construir duas versões do Bootstrap a partir de um código-base.

  2. Em segundo lugar, renomeamos um punhado de classes direcionais para adotar uma abordagem de propriedades lógicas. Most of you have alreadA maioria de vocês já interagiu com propriedades lógicas graças aos nossos utilitários flex - eles substituem as propriedades de direção como left e right em favor das propriedades start e end. Isso torna os nomes e valores das classes apropriados para LTR e RTL sem qualquer sobrecarga.

Por exemplo, em vez de .ml-3 para margin-left, use .ms-3.

Trabalhar com RTL, por meio de nosso Sass de origem ou CSS compilado, não deve ser muito diferente de nosso LTR padrão.

Personalize a partir da fonte

Quando se trata de customização, a maneira preferida é aproveitar as vantagens de usar variáveis, mapas e mixins. Essa abordagem funciona da mesma forma para RTL, mesmo que seja pós-processada a partir dos arquivos compilados, graças ao funcionamento do RTLCSS.

Valores RTL Personalizados

Usando diretivas de valor RTLCSS, você pode fazer de uma saída variável um valor diferente para RTL. Por exemplo, para diminuir o peso de $font-weight-bold ao longo da base de código, você pode usar a sintaxe /*rtl: {value}*/: 

$font-weight-bold: 700 #{/* rtl:600 */} !default;

O que resultaria no seguinte, para nosso CSS padrão e CSS RTL: 

/* bootstrap.css */
dt {
  font-weight: 700 /* rtl:600 */;
}

/* bootstrap.rtl.css */
dt {
  font-weight: 600;
}

Pilha de Fontes Alternativas

No caso de você estar usando uma fonte personalizada, esteja ciente de que nem todas as fontes suportam o alfabeto não latino. Para mudar da família pan-europeia para a árabe, pode ser necessário usar /*rtl:insert: {value}*/ em sua pilha de fontes para modificar os nomes das famílias de fontes.

Por exemplo, para mudar de Helvetica Neue Webfont do LTR para Helvetica Neue Arabic do RTL, seu código Sass irá se parecer com este: 

$font-family-sans-serif:
  Helvetica Neue #{"/* rtl:insert:Arabic */"},
  // Família de fontes genéricas multiplataforma (fonte padrão da interface do usuário)
  system-ui,
  // Safari para macOS e iOS (San Francisco)
  -apple-system,
  // Chrome < 56 para macOS (San Francisco)
  BlinkMacSystemFont,
  // Windows
  "Segoe UI",
  // Android
  Roboto,
  // Fallback básico da web
  Arial,
  // Linux
  "Noto Sans",
  // Sans serif fallback
  sans-serif,
  // Fontes Emoji
  "Apple Color Emoji", "Segoe UI Emoji", "Segoe UI Symbol", "Noto Color Emoji" !default;

LTR e RTL ao mesmo tempo

Precisa de LTR e RTL na mesma página? Graças ao String Maps do RTLCSS, isso é bastante simples. Envolva seu @import com uma classe e determinada regra de renomeação personalizada para RTLCSS: 

/* rtl:begin:options: {
  "autoRename": true,
  "stringMap":[
    "name": "ltr-rtl",
    "priority": 100,
    "search": ["ltr"],
    "replace": ["rtl"],
    "options": {
      "scope": "*",
      "ignoreCase": false
    }
  ]
} */
.ltr {
  @import "../node_modules/bootstrap/scss/bootstrap";
}
/*rtl:end:options*/

Depois de executar o Sass, em seguida, RTLCSS, cada seletor em seus arquivos CSS será preparado para .ltr, e .rtl para arquivos RTL. Agora você pode usar os dois arquivos na mesma página e simplesmente usar .ltr ou .rtl nos contêiners dos seus componentes para usar uma ou outra direção.

Casos Extremos e Limitados

Embora essa abordagem seja compreensível, preste atenção ao seguinte:

  1. Quando alternar entre .ltr e .rtl, certifique-se de adicionar os atributos dir e lang compatíveis.
  2. Carregar ambos os arquivos pode ser um verdadeiro gargalo de desempenho: considere alguns pontos de otimização, e talvez tentar carregar um desses arquivos de forma assíncrona.
  3. O aninhamento de estilos dessa forma evitará que nosso mixin form-validation-state() funcione conforme o esperado, portanto, é necessário ajustá-lo um pouco sozinho. Veja o Issue #31223.

O caso do Breadcrumb

O separador breadcrumb é o único caso que requer sua própria variável — isto é, $breadcrumb-divider-flipped — padronizando para $breadcrumb-divider.
