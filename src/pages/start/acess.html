Accessibilidade

Uma breve visão geral dos recursos e limitações do Bootstrap para a criação de conteúdo acessível.

O Bootstrap oferece uma estrutura fácil de usar de estilos prontos, ferramentas de layout e componentes interativos, permitindo que os desenvolvedores criem sites e aplicativos visualmente atraentes, funcionalmente ricos e acessível fora da caixa.

Visão Geral e limitação

A acessibilidade geral de qualquer projeto construído com Bootstrap depende em grande parte da marcação do autor, estilo adicional e script que eles incluíram. No entanto, desde que foram implementados corretamente, deve ser perfeitamente possível criar sites e aplicativos com Bootstrap que atendam os padrões WCAG 2.1 (A/AA/AAA), Section 508, e outros padrões e requisitos de acessibilidade semelhantes.

Marcação estrutural

O estilo e o layout do Bootstrap podem ser aplicado numa ampla gama de estruturas de marcação. Essa documentação visa fornecer aos desenvolvedores as melhores práticas com exemplos para demonstrar o uso do Bootstrap em si e ilustrar a marcação semântica apropriada, incluindo maneiras em que potenciais preocupações de acessibilidade podem ser abordadas.

Componentes Interativos

Os componentes interativos do Bootstrap - como diálogos modais, menus suspensos e dicas de ferramentas personalizadas - são corrigidos para funcionar para usuários de toque, mouse e teclado. Por meio do uso de funções e atributos WAI-ARIA, esses componentes também devem ser compreensíveis e operáveis usando tecnologias assistivas (como leitores de tela).

Como os componentes do Bootstrap são propositadamente projetados para serem bastante genéricos, os autores podem precisar incluir mais regras e atributos ARIA, bem como o comportamento JavaScript, para transmitir com mais precisão a natureza e funcionalidade precisas de seu componente. Isso geralmente é disposto na documentação.

Contraste de Cor

Algumas combinações de cores que atualmente compõem a paleta padrão do Bootstrap — usadas em toda a estrutura para coisas como variações de botões, variações de alerta, indicadores de validação de formulários — podem levar a contraste de cored insuficiente (abaixo da recomendação da taxa de contraste de cor textual WCAG 2.1 de 4.5:1 e a relação de contraste de cor não textual WCAG 2.1 de 3:1), particularmente quando usado em um fundo claro. Os autores são encorajados a testar seus usos específicos de cor e, quando necessário, modificar/estender manualmente essas cores padrão para garantir razões adequadas de contraste de cores.

Conteúdo Visualmente Oculto

O conteúdo que deve ser visualmente oculto, mas permanecerá acessível a tecnologias assistivas, como leitores de tela, pode ser estilizado usando uma classe .visually-hidden. Isso pode ser útil em situações em que informações visuais adicionais ou pistas (com significado denotado através do uso de cor) também precisam ser transmitidas para usuários não visuais. 

<p class="text-danger">
  <span class="visually-hidden">Danger: </span>
  Esta ação não é reversível
</p>

Para controles interativos visualmente ocultos, como os links tradicionais de “pular”, use a classe .visually-hidden-focusable. Isso garantirá que o controle se torne visível quando focalizado (para usuários de teclado míope). Cuidado, comparado com as classes equivalente das versões anteriores do Bootstrap 5 (.sr-only e .sr-only-focusable) a classe .visually-hidden-focusable é autônoma, e não deve ser usado em combinação com a classe .visually-hidden. 

<a class="visually-hidden-focusable" href="#content">Skip to main content</a>

Movimento reduzido

O Bootstrap inclui suporte ao recurso de mídia prefers-reduced-motion. Em navegadores/ ambientes que permitem ao usuário especificar sua preferência para movimento reduzido, a maioria dos efeitos de transição CSS no Bootstrap (por exemplo, quando uma caixa de diálogo modal é aberta ou fechada, ou a animação deslizante em carrosséis) é desativada e animações relevantes (como spinners) serão mais lentos.

Em navegadores compatíveis com o prefers-reduced-motion, e quando o usuário não especificou explicitamente que preferiria o movimento reduzido (ou seja, é especificado: prefers-reduced-motion: no-preference), o Bootstrap permite uma rolagem suave usando a propriedade scroll-behavior.
