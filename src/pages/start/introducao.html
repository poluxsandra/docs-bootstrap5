Introdução

Comece com Bootstrap, a estrutura mais popular do mundo para a criação de sites responsivos para dispositivos móveis, com jsDelivr e uma página inicial de modelo.

Começo Rápido

Quer adicionar rapidamente o Bootstrap ao seu projeto? Use jsDelivr, um CDN de código aberto gratuito. Está usando um gerenciador de pacotes ou precisa baixar os arquivos fonte? Vá para a página de download.

CSS

Copie e cole nossa folha de estilo usando o <link> em seu <head> antes de todas as outras, para carregar nosso CSS. 

<link href="https://cdn.jsdelivr.net/npm/bootstrap@5.0.0-beta2/dist/css/bootstrap.min.css" rel="stylesheet" integrity="sha384-BmbxuPwQa2lc/FVzBcNJ7UAyJxM6wuqIj61tLrc4wSX0szH/Ev+nYRRuWlolflfl" crossorigin="anonymous">

JavaScript

Muitos dos nossos componentes usa arquivos JavaScript para funcionar. Especificamente, eles são nossos plug-ins próprios de JavaScript e Popper. Coloque um dos seguintes <script> 's próximo do final da página antes da tag de fechamento </body>, para ativa-los.

    Compilado

    Inclui todos os plugins e dependências do Bootstrap JavaScript com um de nossos dois pacotes. Ambos bootstrap.bundle.js e bootstrap.bundle.min.js incluem o Popper para nossas ferramentas e popovers. Para obter mais informações sobre o que está incluído no Bootstrap, consulte nossa seção de Conteúdo. 

    <script src="https://cdn.jsdelivr.net/npm/bootstrap@5.0.0-beta2/dist/js/bootstrap.bundle.min.js" integrity="sha384-b5kHyXgcpbZJO/tY9Ul7kGkf1S0CWuKcCD38l8YkeH8z8QjE0GmW1gYU5S9FOnJ0" crossorigin="anonymous"></script>

    Separados

    Se você decidir ir com a solução de scripts separados, Popper deve vir primeiro (se você estiver usando dicas de ferramentas ou popovers) e, em seguida, nossos plug-ins JavaScript. 

    <script src="https://cdn.jsdelivr.net/npm/@popperjs/core@2.6.0/dist/umd/popper.min.js" integrity="sha384-KsvD1yqQ1/1+IA7gi3P0tyJcT3vR+NdBTt13hSJ2lnve8agRGXTTyNaBYmCR/Nwi" crossorigin="anonymous"></script>
<script src="https://cdn.jsdelivr.net/npm/bootstrap@5.0.0-beta2/dist/js/bootstrap.min.js" integrity="sha384-nsg8ua9HAw1y0W1btsyWgBklPnCUAFLuTMS2G72MMONqmOymq585AcH49TLBQObG" crossorigin="anonymous"></script>

    Módulos

    Se você usa <script type="module">, por favor, consulte nossa seção usando Bootstrap como um módulo.

Modelo Inicial

Certifique-se de ter suas páginas configuradas com os padrões de design e desenvolvimento mais recentes. Isso significa usar um doctype HTML5 e incluir uma metatag de janela de visualização para comportamento responsivos adequados. Reunindo tudo isso, suas páginas devem ficar assim: 

<!doctype html>
<html lang="en">
  <head>
    <!-- Metatags requiridas -->
    <meta charset="utf-8">
    <meta name="viewport" content="width=device-width, initial-scale=1">

    <!-- Bootstrap CSS -->
    <link href="https://cdn.jsdelivr.net/npm/bootstrap@5.0.0-beta2/dist/css/bootstrap.min.css" rel="stylesheet" integrity="sha384-BmbxuPwQa2lc/FVzBcNJ7UAyJxM6wuqIj61tLrc4wSX0szH/Ev+nYRRuWlolflfl" crossorigin="anonymous">

    <title>Hello, world!</title>
  </head>
  <body>
    <h1>Hello, world!</h1>

    <!-- JavaScript opcional; escolha uma das duas alternativas! -->

    <!-- Opção 1: Bootstrap Compilado com Pupper -->
    <script src="https://cdn.jsdelivr.net/npm/bootstrap@5.0.0-beta2/dist/js/bootstrap.bundle.min.js" integrity="sha384-b5kHyXgcpbZJO/tY9Ul7kGkf1S0CWuKcCD38l8YkeH8z8QjE0GmW1gYU5S9FOnJ0" crossorigin="anonymous"></script>

    <!-- Opção 2: Popper and Bootstrap JS separados -->
    <!--
    <script src="https://cdn.jsdelivr.net/npm/@popperjs/core@2.6.0/dist/umd/popper.min.js" integrity="sha384-KsvD1yqQ1/1+IA7gi3P0tyJcT3vR+NdBTt13hSJ2lnve8agRGXTTyNaBYmCR/Nwi" crossorigin="anonymous"></script>
    <script src="https://cdn.jsdelivr.net/npm/bootstrap@5.0.0-beta2/dist/js/bootstrap.min.js" integrity="sha384-nsg8ua9HAw1y0W1btsyWgBklPnCUAFLuTMS2G72MMONqmOymq585AcH49TLBQObG" crossorigin="anonymous"></script>
    -->
  </body>
</html>

Isso é tudo que você precisa para os requisitos gerais da página. Visite os documentos sobre Layout ou nossos exemplos oficiais para começar a definir o conteúdo e os componentes do seu site.

Globais Importantes

O Bootstrap emprega vários estilos e configurações globais importantes dos quais você precisa estar ciente ao usá-lo, todos eles quase que exclusivamente voltados para a normalização de estilos entre navegadores. Vamos mergulhar.

HTML5 Doctype

O bootstrap requer o uso do Doctype HTML5. Sem ele, você verá algum estilo incompleto descabido, mas incluí-lo não deve causar nenhuma complicação. 

<!doctype html>
<html lang="en">
  ...
</html>

Metatag responsiva

O Bootstrap é desenvolvido usando a filosofia mobile first , uma estratégia em que otimizamos o código para dispositivos móveis primeiro e, em seguida, aumentamos os componentes conforme necessário através de consultas de mídia CSS. Para garantir a renderização e o zoom de toque adequado para todos os dispositivos, é necessário acrescentar uma metatag de janela de visualização de responsividade no seu <head>. 

<meta name="viewport" content="width=device-width, initial-scale=1">

Você pode ver um exemplo disso em ação no Modelo Inicial.

Box-sizing

Para um dimensionamento mais direto em CSS, mudamos o valor global do box-sizing, de content-box para border-box. Isso garante um padding que não afete a largura final de um elemento, mas pode causar problemas com alguns softwares de terceiros, como Google Maps e Google Custom Search Engine.

Nas raras ocasiões em que você precisar substituí-lo, use algo como o seguinte: 

.selector-for-some-widget {
  box-sizing: content-box;
}

Com o snippet acima, os elementos aninhados - incluindo o conteúdo gerado via ::before e ::after —herdarão as especificações do box-sizing através do .selector-for-some-widget.

Saiba mais sobre o modelo de caixa e dimensionamento em CSS Tricks.

Reinício

Para melhorar a renderização entre navegadores, nós usamos a correção de Reinício para corrigir inconsistências entre navegadores e dispositivos, enquanto fornecemos redefinições um pouco mais opinativas para elementos HTML comuns.
