JavaScript

Dê vida ao Bootstrap com nossos plug-ins opcionais de JavaScript. Saiba mais sobre cada plug-in, nossos dados e opções de API programática e muito mais.

Individual ou Compilado

Plugins podem ser incluídos individualmente (usando o Bootstrap individual js/dist/*.js), ou todos de uma vez usando o bootstrap.js ou na forma minificada bootstrap.min.js (não incluir ambos).

Se você usar um bundler (Webpack, Rollup…), poderá usar arquivos /js/dist/*.js que estão prontos para UMD.

Usando Bootstrap Como um Módulo

Nós fornecemos uma versão do Bootstrap construído como ESM (bootstrap.esm.js e bootstrap.esm.min.js) que permite que você use o Bootstrap como um módulo em seu navegador, se houver suporte pelo navegador. 

<script type="module">
  import { Toast } from 'bootstrap.esm.min.js'

  Array.from(document.querySelectorAll('.toast'))
    .forEach(toastNode => new Toast(toastNode))
</script>

Plugins Incompatíveis

Alguns plugins como Dropdown, Tooltip e Popover não aceitam ser usados dentro de um elemento <script> com o atributo module, porque eles são dependentes do Popper. Para obter mais informações sobre o problema clique aqui.

Dependências

Alguns plug-ins e componentes CSS dependem de outros plug-ins. Se você incluir plug-ins individualmente, deverá verificar essas dependências nos documentos.

Nossos menus suspensos, popovers e dicas de ferramentas também dependem do Popper.

Ainda deseja usar o jQuery? É possível!

O Bootstrap 5 foi projetado para ser usado sem jQuery, mas ainda é possível usar nossos componentes com jQuery. Se o Bootstrap detectar jQuery no Window Objeto ele adicionará todos os nossos componentes no sistema de plug-ins do jQuery; isso significa que você pode fazer $('[data-bs-toggle="tooltip"]').tooltip() para habilitar as dicas de ferramentas. O mesmo vale para os nossos outros componentes.

Atributos de arquivos

Quase todos os plug-ins do Bootstrap podem ser ativados e configurados apenas por meio de HTML com atributos de dados (nossa forma preferida de usar a funcionalidade JavaScript). Certifique-se de usar apenas um conjunto de atributos de dados em um único elemento (por exemplo, você não pode acionar uma dica de ferramenta e modal a partir do mesmo botão).

Seletores

Atualmente, para consultar os elementos DOM, competente os métodos nativos querySelector e querySelectorAll por motivos de desempenho, portanto, você deve usar seletores válidos. Se você usar seletores especiais como, por exemplo: collapse:Example certifique-se de escapar deles.

Eventos

Bootstrap fornece eventos personalizados para ações exclusivas na maioria dos plugins. Geralmente, eles são especificados na sua forma infinitiva ou no particípio - onde o infinitivo (ex. show) é acionado no início de um evento, e o particípio (ex. shown) é acionado na conclusão de uma ação.

Todos os eventos infinitivos fornecem a funcionalidade preventDefault(). Isso fornece a capacidade de parar a execução de uma ação antes dela começar. Caso haja retorno falso em um manipulador de eventos, este também chamará automaticamente preventDefault(). 

var myModal = document.getElementById('myModal')

myModal.addEventListener('show.bs.modal', function (event) {
    if (!data) {
      return event.preventDefault() // impede modal de ser mostrado(shown)
  }
})

Eventos jQuery

Bootstrap detectará jQuery se ele estiver no Window Objeto e não houver o atributo data-bs-no-jquery especificado no elemento <body>. Se jQuery for encontrado, Bootstrap emitirá eventos graças ao sistema de eventos do jquery. Então se você quiser colocar eventos em escuta, deverá usar os métodos do jQuery, que são:(.on, .one) em vez do addEventListener. 

$('#myTab a').on('shown.bs.tab', function () {
  // do something...
})

API Programática

Todos os construtores aceitam um objeto de opções opcionais ou nada (que inicia um plugin com seu comportamento padrão): 

var myModalEl = document.getElementById('myModal')

var modal = new bootstrap.Modal(myModalEl) // initialized with defaults
var modal = new bootstrap.Modal(myModalEl, { keyboard: false }) // initialized with no keyboard

Se você gostaria de obter uma instância de plugin particular, cada plugin expõe um método getInstance. Para recuperá-lo diretamente de um elemento, faça o seguinte: bootstrap.Popover.getInstance(myPopoverEl).

Funções assíncronas e transições

Todos os método da API Programática são assíncronas e retorna assim que o chamador inicia a transição, mas antes que ela termine.

Para executar uma ação após a conclusão da transição, você pode ouvir o evento correspondente. 

var myCollapseEl = document.getElementById('#myCollapse')

myCollapseEl.addEventListener('shown.bs.collapse', function (event) {
  // Ação que é executada quando a área dobrável é expandida
})

Além disso, uma chamada de método em um componente de transição será ignorada. 

var myCarouselEl = document.getElementById('myCarousel')
var carousel = bootstrap.Carousel.getInstance(myCarouselEl) // Recuperar uma instância do Carrossel

myCarouselEl.addEventListener('slid.bs.carousel', function (event) {
  carousel.to('2') // Deslizará para o slide 2 assim que a transição para slide 1 estiver concluída
})

carousel.to('1') // Vai começar a deslizar para o slide 1 e retorna para o chamador
carousel.to('2') // !! Será ignorado, pois a transição para o slide 1 não está concluída !!

Configurações Padrão

Você pode alterar as configurações padrão de um plugin modificando o objeto Constructor.Default do plugin: 

// alteração da opção "Padrão" plugin Modal do "teclado" para "false"
bootstrap.Modal.Default.keyboard = false

Sem Conflito (somente se você usar jQuery)

Às vezes, é necessário usar plug-ins Bootstrap com outras estruturas de IU. Nessas circunstâncias, podem ocorrer ocasionalmente conflitos de namespace. Se isso acontecer, você pode chamar .noConflict no plugin do qual deseja reverter o valor. 

var bootstrapButton = $.fn.button.noConflict() // retorna o valor $.fn.button previamente atribuído
$.fn.bootstrapBtn = bootstrapButton // dar ao $().bootstrapBtn a funcionalidade Bootstrap

Números das Versões

A versão de cada um dos plug-ins do Bootstrap pode ser acessado por meio da propriedade VERSION do construtor do plugin. Por exemplo para o plugin Utilitários(tooltip): 

bootstrap.Tooltip.VERSION // => "5.0.0-beta2"

Sem Fallbacks Especiais quando o JavaScript está desativado

Os plug-ins do Bootstrap não possuem fallbacks quando o JavaScript está desabilitado. Se você se preocupa com a experiência do usuário neste caso, use o elemento <noscript> para explicar a situação (e como reativar o JavaScript) para seus usuários e / ou adicionar seus próprios fallbacks personalizados.

Bibliotecas de Terceiros

Bootstrap não suporta oficialmente bibliotecas JavaScript de terceiros como Prototype ou jQuery UI. Apesar do .noConflict e eventos namespace, pode haver problemas de compatibilidade que você precisa corrigir por conta própria.

Sanitizador

Utilitários e o Popovers usam nosso sanotizador integrado para limpar opções que aceitam HTML.

O valor padrão allowList é o seguinte: 

var ARIA_ATTRIBUTE_PATTERN = /^aria-[\w-]*$/i
var DefaultAllowlist = {
  // Atributos globais permitidos em qualquer elemento fornecido abaixo.
  '*': ['class', 'dir', 'id', 'lang', 'role', ARIA_ATTRIBUTE_PATTERN],
  a: ['target', 'href', 'title', 'rel'],
  area: [],
  b: [],
  br: [],
  col: [],
  code: [],
  div: [],
  em: [],
  hr: [],
  h1: [],
  h2: [],
  h3: [],
  h4: [],
  h5: [],
  h6: [],
  i: [],
  img: ['src', 'srcset', 'alt', 'title', 'width', 'height'],
  li: [],
  ol: [],
  p: [],
  pre: [],
  s: [],
  small: [],
  span: [],
  sub: [],
  sup: [],
  strong: [],
  u: [],
  ul: []
}

Se quiser adicionar novos valores ao padrão allowList você pode fazer o seguinte: 

var myDefaultAllowList = bootstrap.Tooltip.Default.allowList

// Para permitir elementos de tabela
myDefaultAllowList.table = []

// Para permitir elementos td e atributos data-bs-option em elementos td
myDefaultAllowList.td = ['data-bs-option']

// Você pode acrescentar com o "push" seu regex personalizado para validar seus atributos.
// Tenha cuidado com expressões regulares relaxadas
var myCustomRegex = /^data-my-app-[\w-]+/
myDefaultAllowList['*'].push(myCustomRegex)

Se você deseja ignorar nosso sanitizador porque prefere usar uma biblioteca dedicada, por exemplo DOMPurify, faça o seguinte: 

var yourTooltipEl = document.getElementById('yourTooltip')
var tooltip = new bootstrap.Tooltip(yourTooltipEl, {
  sanitizeFn: function (content) {
    return DOMPurify.sanitize(content)
  }
})
