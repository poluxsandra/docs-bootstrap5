Navegadores e dispositivos

Aprenda sobre os navegadores e dispositivos, dos modernos aos antigos, que são permitidos pelo Bootstrap, incluindo peculiaridades e bugs conhecidos para cada um.

Navegadores Suportados

O bootstrap possui suporte para as versões mais recentes e estáveis dos principais navegadores e plataformas.

Navegadores alternativos que usam a versão mais recente do WebKit, Blink ou Gecko, seja diretamente ou por meio da API de visualização da plataforma, não são explicitamente permitidos. No entanto, o Bootstrap pode (na maioria dos casos) ser aplicado e funcionar corretamente nesses navegadores também. Informações de suporte mais específicas são fornecidas abaixo.

Você pode encontrar nosso suporte aos navegadores com suas versões no .browserslistrc file: 

# https://github.com/browserslist/browserslist#readme

>= 0.5%
last 2 major versions
not dead
Chrome >= 60
Firefox >= 60
# needed since Legacy Edge still has usage; 79 was the first Chromium Edge version
# should be removed in the future when its usage drops or when it's moved to dead browsers
not Edge < 79
Firefox ESR
iOS >= 10
Safari >= 10
Android >= 6
not Explorer <= 11

Nós usamos Autoprefixador para lidar com o suporte pretendido do navegador por meio de prefixos CSS, que usa Browserslist para gerenciar essas versões de navegador. Consulte sua documentação para saber como integrar essas ferramentas em seus projetos.

Dispositivos Móveis

De modo geral, o Bootstrap suporta as versões mais recentes dos navegadores padrão de cada plataforma principal. Observe que os navegadores proxy (como Opera Mini, modo Turbo do Opera Mobile, UC Browser Mini, Amazon Silk) não são suportados.

Desktops

Da mesma forma, as versões mais recentes da maioria dos navegadores de desktop são suportadas.

Chrome Firefox Microsoft Edge Opera Safari
Mac Suportado Suportado Suportado Suportado Suportado
Windows Suportado Suportado Suportado Suportado —

Para o Firefox, além da versão estável normal mais recente, também oferecemos suporte para a versão mais recente do Extended Support Release (ESR) versão do Firefox.

Extraoficialmente, o Bootstrap deve se comportar bem o suficiente no Chromium, Chrome e Firefox para Linux, embora não sejam oficialmente suportados.

Internet Explorer

O Internet Explorer não é compatível. Se você precisar de suporte ao Internet Explorer, use o Bootstrap v4.

Modais e Dropdowns em dispositivos móveis

Overflow e Barra de Rolagem

O suporte ao overflow:hidden; no elemento <body> é bastante limitado no Android e iOS. Para esse fim, quando você rola a parte superior ou inferior de um modal em qualquer um dos navegadores desses dispositivos, o conteúdo do <body> começará a rolar. Consulte o Chrome Bug #175502 (corrigido pelo Chrome v40) e o WebKit Bug #153852.

Campos de texto e rolagem do iOS

A partir do iOS 9.2, enquanto um modal está aberto, se o toque inicial de um gesto de rolagem estiver dentro dos limites de um textual no elemento <input> ou no elemento <textarea>, o conteúdo do <body> abaixo do modal será rolado em vez do próprio modal. Consulte o WebKit Bug #153856.

Navbars Suspensas (Dropdowns)

O elemento .dropdown-backdrop não é usado no iOS no navegador por causa da complexidade do z-indexing. Assim, para fechar os menus suspensos nas barras de navegação, você deve clicar diretamente no elemento suspenso (ou qualquer outro elemento que irá disparar um evento de clique no iOS).

Zoom dos Navegadores

O zoom da página inevitavelmente apresenta problemas de renderização em alguns componentes, tanto no Bootstrap quanto no resto da web. Dependendo do problema, podemos consertá-lo (pesquise primeiro e, em seguida, abra um problema, se necessário). No entanto, tendemos a ignorá-los, pois geralmente não têm solução direta além de soluções alternativas.

Validadores

A fim de fornecer a melhor experiência possível para navegadores antigos e com erros, o Bootstrap usa hacks de navegador CSS em vários lugares para direcionar CSS especial para certas versões de navegador, a fim de contornar bugs nos próprios navegadores. Esses hacks fazem com que os validadores CSS se queixem de que são inválidos. Em alguns lugares, também usamos recursos CSS de ponta que ainda não estão totalmente padronizados, mas são usados exclusivamente para aprimoramento progressivo.

Esses avisos de validação não importam na prática, uma vez que a parte "não hacky" de nosso CSS é validada totalmente e as partes "hacky" não interferem no funcionamento adequado da parte "não hacky", por isso ignoramos esses avisos específicos.

Nossos documentos HTML também têm alguns avisos de validação de HTML triviais e inconseqüentes devido à nossa inclusão de uma solução alternativa para um certo Firefox Bug.
