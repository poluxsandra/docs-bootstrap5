Breakpoints

Breakpoints são larguras personalizáveis ​​que determinam como seu layout responsivo se comporta em dispositivos ou tamanhos de janela de exibição no Bootstrap.

Conceitos Centrais

  • Os Breakpoints são os blocos de construção do design responsivo. Use-os para controlar quando seu layout pode ser adaptado em uma janela de visualização ou tamanho de dispositivo específico.

  • Use consultas de mídia para arquitetar seu CSS por Breakpoint. As consultas de mídia são um recurso do CSS que permite aplicar condicionalmente estilos com base em um conjunto de parâmetros do navegador e do sistema operacional. Nós mais comumente usamos min-width em nossas consultas de mídia.

  • A meta é o design responsivo e mobile first. O CSS do Bootstrap visa aplicar o mínimo de estilos para fazer um layout funcionar no menor Breakpoint e, em seguida, camadas em estilos para ajustar esse design para dispositivos maiores. Isso otimiza seu CSS, melhora o tempo de renderização e fornece uma ótima experiência para seus visitantes.

Breakpoints disponíveis

O bootstrap inclui seis Breakpoints padrão, às vezes chamados de camadas de grade , para construção responsiva. Esses Breakpoints podem ser personalizados se você estiver usando nossos arquivos-fonte Sass.

Breakpoint Infixo de Classe Dimensões
X-Small(Muito Pequeno) Nenhum <576px
Small(Pequeno) sm ≥576px
Medium(Médio) md ≥768px
Large(Grande) lg ≥992px
Extra large(Muito Grande) xl ≥1200px
Extra extra large(Gigante) xxl ≥1400px

Cada Breakpoint foi escolhido para acomodar confortavelmente contêineres cujas larguras são múltiplos de 12. Os pontos de interrupção também são representativos de um subconjunto de tamanhos de dispositivos comuns e dimensões de janela de visualização - eles não visam especificamente cada caso de uso ou dispositivo. Em vez disso, os intervalos fornecem uma base sólida e consistente para construir em quase todos os dispositivos.

Esses pontos de interrupção são personalizáveis ​​por meio do Sass - você os encontrará em um mapa do Sass em nossa folha de estilo _variables.scss. 

$grid-breakpoints: (
        xs: 0,
        sm: 576px,
        md: 768px,
        lg: 992px,
        xl: 1200px,
        xxl: 1400px
      );

Para obter mais informações e exemplos sobre como modificar nossos mapas e variáveis ​​Sass, consulte a seção Sass da documentação do Grid..

Consultas de Mídia

Como o Bootstrap foi desenvolvido para ser mobile first, usamos um punhado de consultas de mídia para criar Breakpoints sensíveis para nossos layouts e interfaces. Esses Breakpoints são baseados principalmente nas larguras mínimas da viewport e nos permitem dimensionar os elementos conforme a viewport muda.

Min-width(Largura Mínima)

O Bootstrap usa principalmente os seguintes intervalos de consulta de mídia - ou Breakpoint - em nossos arquivos-fonte Sass para nosso layout, sistema de grade e componentes. 

// Mixins da Fonte
      
      // Nenhuma consulta de mídia necessária para o ponto de interrupção xs, pois é eficaz "@media (min-width: 0) { ... }"
      @include media-breakpoint-up(sm) { ... }
      @include media-breakpoint-up(md) { ... }
      @include media-breakpoint-up(lg) { ... }
      @include media-breakpoint-up(xl) { ... }
      @include media-breakpoint-up(xxl) { ... }
      
      // Uso
      
      // Exemplo: ocultar um elemento começando em "min-width: 0" e, em seguida, mostrar no ponto de interrupção "sm" 
      .custom-class {
        display: none;
      }
      @include media-breakpoint-up(sm) {
        .custom-class {
          display: block;
        }
      }

Esses mixins Sass são traduzidos em nosso CSS compilado usando os valores declarados em nossas variáveis ​​Sass. Por exemplo: 

// Dispositivos X-Small (modo retrato em celulares, menor que 576px)
      // Nenhuma consulta de mídia para "xs", pois este é o padrão no Bootstrap
      
      // Dispositivos Small (modo paisagem em telefones, 576px ou maior)
      @media (min-width: 576px) { ... }
      
      // Dispositivos Medium (tablets, 768px ou maior)
      @media (min-width: 768px) { ... }
      
      // Dispositivos Large (desktops, 992px ou maior)
      @media (min-width: 992px) { ... }
      
      // Dispositivos X-Large (desktops largos, 1200px ou maior)
      @media (min-width: 1200px) { ... }
      
      // Dispositivos XX-Large (desktops largos, 1400px ou maior)
      @media (min-width: 1400px) { ... }

Max-width(Largura Máxima)

Ocasionalmente, usamos consultas de mídia que vão na outra direção (o tamanho de tela fornecido ou menor): 

// Nenhuma consulta de mídia necessária para o ponto de interrupção "xs", pois é eficaz  "@media (max-width: 0) { ... }"
      @include media-breakpoint-down(sm) { ... }
      @include media-breakpoint-down(md) { ... }
      @include media-breakpoint-down(lg) { ... }
      @include media-breakpoint-down(xl) { ... }
      @include media-breakpoint-down(xxl) { ... }
      
      // Exemplo: estilo de Breakpoint médio e menor
      @include media-breakpoint-down(md) {
        .custom-class {
          display: block;
        }
      }

Esses mixins pegam esses Breakpoint declarados, subtraem .02px deles e os usam como nossos valores de max-width. Por exemplo: 

// X-Small (modo retrato em celulares, menor que 576px)
      @media (max-width: 575.98px) { ... }
      
      // Dispositivos Small (modo paisagem em celulares, menor que 768px)
      @media (max-width: 767.98px) { ... }
      
      // Dispositivos Medium (tablets, menor que 992px)
      @media (max-width: 991.98px) { ... }
      
      // Dispositivos Large (desktops, menor que 1200px)
      @media (max-width: 1199.98px) { ... }
      
      // Dispositivos X-Large(large desktops, menor que 1400px)
      @media (max-width: 1399.98px) { ... }
      
      // Dispositivos XX-Large (larger desktops)
      // Sem consulta de mídia, pois o Breakpoint xxl não tem limite superior em sua largura
Por que subtrair 0,02 px? Navegadores atualmente não suportam consultas de contexto gama, então contornamos as limitações dos prefixos min- and max- prefixes e viewports com larguras fracionários (que pode ocorrer sob certas condições em dispositivos high-dpi, por exemplo), usando os valores com maior precisão.

Breakpoint Único

Existem também consultas de mídia e mixins para direcionar um único segmento de tamanhos de tela usando as larguras de Breakpoint mínima e máxima. 

@include media-breakpoint-only(xs) { ... }
      @include media-breakpoint-only(sm) { ... }
      @include media-breakpoint-only(md) { ... }
      @include media-breakpoint-only(lg) { ... }
      @include media-breakpoint-only(xl) { ... }
      @include media-breakpoint-only(xxl) { ... }

Por exemplo, o @include media-breakpoint-only(md) { ... } resultará em : 

@media (min-width: 768px) and (max-width: 991.98px) { ... }

Entre Breakpoints

Da mesma forma, as consultas de mídia podem abranger várias larguras de Breakpoints:

@include media-breakpoint-between(md, xl) { ... }

O que resulta em: 

// Exemplo
      // Aplicado estilos desde dispositivos médios até dispositivos extragrandes 
      @media (min-width: 768px) and (max-width: 1199.98px) { ... }
